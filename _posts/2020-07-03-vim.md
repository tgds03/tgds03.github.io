## Normal Mode 커맨드 원리

 Normal Mode의 수많은 커맨드들은 숫자와 더불어 크게 다음 3가지로 분류할 수 있다.
 
- 수량자 (#) : 명령어를 반복할 횟수를 지정할 수 있다. 생략할 수 있으며, 이 때 default 값은 1이다. 동사와 명사 앞에 올 수 있다.
- 동사 (V) : 실행할 명령의 동작을 의미한다.
	- 단독 동사 : 명사와 조합되지 않는 동사들을 말한다.
- 명사 (N) : 명령어로 지정하는 대상을 의미한다.
	- 이동 명령 : 커서로 이동하는 범위에 있는 텍스트들을 대상으로 지정한다.
	- 텍스트 개체 : 조건에 맞는 범위의 텍스트들을 대상으로 지정한다.

 그리고 (단독 동사를 제외한) 동사와 명사는 `VN` 꼴로 묶어, 한 명령어처럼 취급할 수 있다. 예를 들어 `2dj`는 현재 줄에서 2줄 밑의 줄까지 제거한다는 의미가 된다. **한 명령어**로 취급하는 것을 강조하는 이유는 단독 동사 `.`를 통해 그 명령어를 반복할 수 있기 때문이다.

### 동사 (Verbs)

 동사에 속하는 명령어들은 다음과 같다.
 
 - `d` (Delete) : 삭제 명령.
 - `c` (Change) : 수정 명령. `d`와 비슷하나 Insert Mode로 진입한다.
 - `>`, `<` (Indent) : 탭 삽입, 제거. (들여쓰기와 내어쓰기)
 - `y` (Yank) : 복사.

 특히 `c`의 경우, Insert Mode 후 Esc를 누를 때까지의 동작들(타이핑)도 한 명령어에 포함된다. 이 덕에 `.`과 같이 일괄 변경 작업에 많이 쓰이게 된다.

 단독 동사와 달리 이들은 키를 한번 눌러도, 명사를 받기 위해서 다시 키 입력을 대기하는 상태가 된다. 이때 명사가 아니라 자기 자신을 다시 받을 수 있는데, 대개 이 경우 행 단위로 명령을 실행하게 된다. 예를 들어, `dd`는 커서가 위치한 행 전체를 제거한다.
 
#### 단독 동사

 단독 동사는 뒤에 올 명사를 취급하지 않는다. 그래서 키를 누르자마자 명령이 실행하게 된다. 위에 서술한 동사들을 제외하면 대부분이 단독 동사이지만, 실제로 쓰이는 정도는 동사들이 훨씬 많다. (`.`은 예외로 전체 명령어 중에서도 가장 많이 쓰이는 정도라고 한다)
 
 - `v`, `V` (Visual select) : 비주얼 모드 진입.
 - `x` : 제거. 커서에 위치한 텍스트 하나를 지운다. 수량자를 지정하면 커서 뒤 #글자까지 지운다.
 - `r` (replace) : 교체. 커서에 위치한 텍스트 하나를 바꾼다. 이때 새로 바꿀 문자를 입력받게 된다. 수량자를 지정하면 커서 뒤의 #글자까지 교체한다.
 - `R` : `r`과 비슷하지만 Insert Mode로 진입하여 원하는 만큼 바꾼다.
 - `p`, `P` : 붙여넣기. 이때 삽입되는 것은 `y`로 복사한 것 뿐만 아니라 **마지막으로 지워진 텍스트**가 될 수도 있다. 각각 커서 앞, 뒤에 삽입한다.

### 명사 (Nouns)

  명사에 속하는 명령어들은 이동 명령과 텍스트 개체로 나뉜다. 여기서 텍스트 개체는 찾는 방식에 따라 다시 나뉘게 된다.
  
#### 이동 명령
 
  편집 중 커서를 이동하는 명령들을 말한다. 명령어를 입력하기 시작한 커서 위치에서부터, 이동하고 난 후의 커서 위치까지의 텍스트들을 대상으로 삼는다. 아래 명령어들이 이에 속한다.
  
  - `h`, `j`, `k`, `l` : 각각 좌,하,상,우 방향으로 커서를 이동한다.
  - `w`, `e`, `b` : 단어별로 이동한다. 각각 다음 단어, 단어의 끝, 이전 단어로 이동한다.
  - `;`, `,` : `f`, `F`, `t`, `T`의 검색 결과를 순회한다. `;`은 정방향, `,`은 역방향으로 순회한다.
  - `n`, `N` : `/`, `?`의 검색 결과를 순회한다. 각각 정방향, 역방향이다. 위와 달리 문서 전체를 순회한다.

#### 범위 텍스트 개체

 명령어의 조건에 맞는 텍스트들을 대상으로 한다. 다음과 같은 명령어들이 있다.
 
 - `iw` (inner word) : 현재 커서가 위치한 단어를 대상으로 한다.
 - `it` (inner tag) : HTML에서, 현재 커서가 위치한 가장 가까운 태그의 컨텐츠를 대상으로 한다.
 - `i(` (inner brackets) : 현재 커서가 위치한 () 안의 부분을 대상으로 한다. {}나 []도 비슷하게 쓸 수 있다.
 - `i"` (inner quotes) : 현재 커서가 위치한 "" 안의 부분을 대상으로 한다.
 - `ip` (inner paragraph) : 현재 커서가 위치한 문단을 대상으로 한다. (코드에서는 스코프가 문단에 대응된다)
 - `as` (a sentence) : 현재 커서가 위치한 문장을 대상으로 한다.

 대부분 공백이나 괄호 등의 구분자 사이에 있는 범위로 지정한다.
 
#### 검색 텍스트 개체

 사용자가 입력한 문자열 패턴 (정규표현식)에 **매칭되는 위치까지의 텍스트**를 대상으로 한다. 다음 명령어들로 매칭을 검색할 수 있다.
 - `f` 또는 `F` : 현재 커서에서부터 매칭되는 문자까지를 대상으로 한다. 검색은 해당 행에서만 이루어진다. 이때 **매칭 문자를 포함**한다.  `F`는 역순으로 매칭 문자를 찾는다. 
 - `t` 또는 `T` : `f`, `F`와 같으나, **매칭 문자를 포함하지 않는다.** 마찬가지로 해당 행에서만 검색한다.
 - `/{term}`, `?{term}` : 각각 `t`와 `T`와 같으나 문자열을 입력할 수 있으며, 문서 전체에서 검색한다.

## Buffer, window
 여타 프로그램의 탭이라 부르는 그게 vim에선 window이고, vim의 탭은 레이아웃에 가깝다. ([#](https://bakyeono.net/post/2015-08-13-vim-tab-madness-translate.html)) 
 
 보통 편집기에서 탭이라고 말하면, 한 파일에 탭 하나씩 대응될 것이란 함축이 있다. (적어도 옛날에는...?) vim의 window를 탭이라고 하지 않고 그렇게 부르는 것은 이 때문이다.
  vim에선 파일 하나를 Buffer라고 부르고, 한 Buffer를 여러 window에 띄우고 작업할 수 있다. 그러니까, 한 파일을 여러 화면에 띄워놓고 동시에 작업할 수도 있다는 얘기이다. 
  또한 여러 window의 구성을 저장하는 것이 탭이 된다.

 Buffer가 파일에 대응되니까... 파일을 열고 닫을 때 쓰던 `:o`, `:w`도 사실 Buffer 명령어였던 셈이다. 그리고 모든 Buffer 명령어들은 Ex mode에서 작동된다.
 
 - `:o` : 파일 열기
 - `:w` : 파일 저장
 - `:buffers` : 현재 열려있는 buffer의 목록을 보여준다.
 - `:#buffer` : 지정된 버퍼를 윈도우에 불러온다.
 - `:bnext` : 다음 버퍼를 윈도우에 불러온다.
 - `:bnew` : 파일을 새 버퍼로 불러들인다. 
 - `:bdelete` : 지정 버퍼를 해제한다.
 - `:ball` : 모든 버퍼를 윈도우로 불러온다.

  `:#bnext`, `:#bnew`, `:#bdelete`들은 모두 숫자를 붙여서 특정 버퍼를 지정할 수 있다.
 `:b <Tap>`으로 `b`로 시작하는 명령어들을 확인할 수 있으며, 대부분의 buffer 명령어는 이렇게 찾아볼 수 있다.

 쓰이는 빈도만 따지면, Buffer보다는 window 명령어들이 더 많이 쓰일 것 같다. window 명령어들도 몇몇은 마찬가지로 `:`을 치고 Ex mode에서 작동하지만, 대부분이 `<Ctrl> W`로 시작한다.
 
 - `:q` : 현재 윈도우를 닫는다.
 - `:sp` : 윈도우를 위 아래로 분할한다.
 - `:vsp` : 윈도우를 좌우로 분할한다.
 - `<Ctrl>-W` + `h`, `j`, `k`, `l` : 좌, 하, 상, 우 에 있는 윈도우로 이동한다.
 - `<Ctrl>-W` + `p` : 마지막으로 있었던 윈도우로 이동한다.
 - `<Ctrl>-W` + `r`, `R` : 버퍼를 우하단 / 좌상단으로 윈도우 한 칸 씩 이동한다.
 - `<Ctrl>-W` + `H`, `J`, `K`, `L` : 현재 윈도우를 좌, 하, 상, 우 끝으로 이동한다.
 - `<Ctrl>-W` + `=` : 모든 윈도우의 높이, 너비를 (거의) 똑같이 맞춘다.
 -  `<Ctrl>-W` + `#+`, `#-` : 현재 윈도우의 높이를 #만큼 증/감한다.
 - `<Ctrl>-W` + `#>`, `#<` : 현재 윈도우의 너비를 #만큼 증/감한다.

 탭에 관한 명령어들은 다음이 있다.
 
 - `:tabs` : 열려 있는 탭 리스트를 보여준다.
 - `:newtab` : 새로운 탭을 추가한다
 - `:tab {window}` : 입력한 윈도우를 새 탭으로 추가한다.
 - `:tabclose`, `:tabc`, `:#tabclose` : 탭을 종료한다.
 - `:tabo`, `:tabonly` : 현재 탭을 제외하고 모두 닫는다.
 - `#gt` : #번째 탭으로 이동한다.	
 - `:tabn`, `:gt` / `:tabp`, `:gT` : 각각 다음 탭, 이전 탭으로 이동한다.
 - `<Ctrl-W>` + `T` : 현재 윈도우를 새 탭으로 이동한다.
